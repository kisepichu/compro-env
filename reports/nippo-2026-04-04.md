|-----------|------|--------------|------|
| 「壁打ち」 | 設計議論 | 適切（業界慣習語） | AI との設計議論をこう呼ぶのは定着している |
| `bash mode` | Claude Code の Bash ツール | 「Bash ツール」または「bash tool」 | Claude Code の機能名は「Bash ツール」が正式 |
| 「骨格 + todo!()」 | 実装戦略 | 「スケルトン実装」または「stub 実装」 | `todo!()` は Rust のマクロ名なので文脈は明確 |
| 「2 段階」 | テスト方法 | 「2 フェーズ」or「2 ステップ」 | いずれでも通じる |

**コミュニケーション改善案**
- Q1〜Q14 形式での回答は AI にとって解析しやすく、設計壁打ちでは有効なパターンだった。継続して活用すると良い
- 「まだ壁打ちを続ける」という継続指示は会話の流れをコントロールするのに効果的だった
- bash mode 障害のデバッグでは、`/tmp` の状態を貼り付けて情報を提供した点が解決への糸口になった。環境情報の具体的な貼り付けは引き続き有効
